## 🚀 Inicio Rápido

### 1. Verificar Setup
```bash
# Verificar Node.js
node --version  # v16 o superior

# Verificar Git
git --version   # v2.30 o superior

# Verificar rama actual
git branch      # Debe ser 'dev'
```

### 2. Instalar Dependencias (cuando tengamos frontend)
```bash
cd frontend
npm install
```

### 3. Variables de Entorno
Crear archivo `.env.local` en la carpeta `frontend/`:
```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key_here
VITE_API_URL=https://your-project.supabase.co/rest/v1
```

### 4. Ejecutar en Desarrollo
```bash
npm run dev
# Acceder a http://localhost:5173
```

## 🌿 Git Workflow (Cómo Trabajar)

### Paso 1: Crear rama para tu feature
```bash
# Actualizar rama dev
git checkout dev
git pull origin dev

# Crear rama nueva desde dev
git checkout -b feature/mi-feature
```

### Paso 2: Hacer cambios y commits
```bash
# Ver cambios
git status

# Agregar cambios
git add .

# Commit descriptivo
git commit -m "feat(operaciones): agregar validación de riesgo máximo"
```

### Tipos de commits permitidos:
- `feat:` Nueva funcionalidad
- `fix:` Corrección de bug
- `docs:` Cambios en documentación
- `style:` Formato (no lógica)
- `refactor:` Cambio de código sin nuevo comportamiento
- `perf:` Mejora de performance
- `test:` Agregar tests

### Paso 3: Push a GitHub
```bash
git push origin feature/mi-feature
```

### Paso 4: Pull Request
1. Ir a GitHub → Pull Requests
2. Crear PR hacia `dev` (no `main`)
3. Describir cambios
4. Esperar revisión

### Paso 5: Merge
Después de aprobación, hacer merge desde GitHub UI o:
```bash
git checkout dev
git pull origin dev
git merge feature/mi-feature
git push origin dev
```

## 🔐 Credenciales & Secretos

**IMPORTANTE:** Nunca commitear credenciales. Usar `.env.local`:

```
# .env.local (no commitear)
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
```

**Guardado seguro:**
1. Pedir credenciales al propietario del proyecto
2. Guardar solo en `.env.local` local
3. Agregar `.env.local` a `.gitignore`

## ❓ FAQ

### P: ¿Dónde pongo mis credenciales?
**R:** En `.env.local` que NO se commitea. Ver sección "Credenciales & Secretos"

### P: ¿Cómo creo una rama?
**R:** `git checkout -b feature/nombre` desde rama `dev`

### P: ¿Cómo hago push?
**R:** `git push origin feature/nombre` y luego Pull Request

### P: ¿Qué rama es producción?
**R:** `main` es producción. Desarrollar en `dev`

### P: ¿Cómo accedo a Supabase?
**R:** Dashboard en https://app.supabase.com (credenciales del owner)

### P: ¿Puedo modificar la BD directamente?
**R:** Mejor usar migraciones. Ver documentación de Supabase.
